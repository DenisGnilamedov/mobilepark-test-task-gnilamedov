**Рекомендации:**
1.	Используем более легкий образ slim с указанием версии для уменьшения размера и стабильности.
2.	Собираем образ в multistage варианте. Это разделит этапы на сборку зависимостей и runtime, таким образом размер образа может существенно уменьшиться.
3.	Меняем порядок выполнения слоев. Часто изменяемые слои размещаем как можно «ниже» для ускорения сборки при изменениях в коде. Сначала ставим зависимости из requirements, потом копируем код. 
4.	Создаем `.dockerignore` и добавляем туда файлы, не нужные для сборки. Содержимое файла ниже.
5.	Оптимизируем выполнение команд установки, чтобы не ставилось ненужное и не оставался мусор в виде кэша:  
   •	`apt-get install -y --no-install-recommends`  
   •	`pip install --no-cache-dir -r /requirements.txt`  
6.	Для запуска приложения создаем non-root юзера для безопасности.
7.	Добавляем в ENV переменные `PYTHONDONTWRITEBYTECODE=1 PYTHONUNBUFFERED=1` – не храним лишний кэш и исключаем побочные эффекты из-за non-root прав. Выключаем буферизацию вывода для более быстрого вывода логов.
